# Bilibili 视频生成网站设计文档

## 1. 项目背景
为了让非专业用户也能快速生成并上传到 B 站的视频，构建一个基于 Web 的一键生成工具。本项目使用 **Next.js** 构建前端与服务端接口，整合 GPT 文本生成、文本转语音(TTS) 以及视频合成服务。

## 2. 项目目标
- 提供简单易用的网页界面，输入主题即可生成视频。
- 集成高质量、价格合理的第三方 API（脚本生成、TTS、视频合成）。
- 支持生成完成后自动上传到 B 站或下载本地。

## 3. 架构概述
```
Browser ──▶ Next.js 前端 ──▶ /api/generate ──▶ 外部视频生成服务
                                   │
                                   └──▶ B 站上传 API
```
- **前端**：Next.js 13 App Router，响应式布局，使用现代 UI 风格。
- **服务端接口**：Next.js API Route `/api/generate`，调用外部 GPT、TTS 及视频合成 API。
- **外部服务**：
  - GPT：生成视频脚本。
  - TTS：根据脚本合成语音。
  - 视频合成：将语音、字幕及素材拼接成成品。
  - B 站 API：上传视频及封面。

## 4. 用户流程
1. 用户在首页输入视频主题、时长等信息。
2. 前端调用 `/api/generate`，后端依次执行脚本生成、TTS、视频合成。
3. 生成完成后返回视频预览地址与上传结果。

## 5. 前端设计
- 主题色：蓝白配色，突出科技感与专业感。
- 布局：居中卡片式表单，顶部展示项目标题，底部显示生成状态。
- 使用 React Hooks 管理状态，使用 CSS Modules 进行样式管理。

## 6. API 设计
`POST /api/generate`
- 请求体：`{ "topic": string }`
- 响应体：`{ "message": string, "videoUrl?: string }`

此接口在当前版本中仅返回模拟数据，后续可替换为真实的外部 API 调用逻辑。

## 7. 后续扩展

- 增加用户登录与个人历史记录管理。
